# 🎯 Quick Reference - 체계적 개발 프로세스

## 필수 단계

### 1️⃣ 분석 및 설계
```
1. 현재 상태 분석 (parse_file)
2. 상세 설계 제시
3. ✋ "이대로 진행해도 될까요?"
```

### 2️⃣ 승인 및 백업
```
1. 사용자 승인 대기
2. Git 백업 생성
3. 작업 시작 알림
```

### 3️⃣ 단계별 수정
```
1. 각 단계 시작 전 알림
2. replace_block으로 수정
3. 진행 상황 보고
```

### 4️⃣ 결과 보고
```
1. 변경된 파일 목록
2. 수정된 함수/클래스 상세
3. Git 커밋 내역
```
