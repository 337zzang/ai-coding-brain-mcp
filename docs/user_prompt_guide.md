
# AI Coding Brain MCP - 유저 프롬프트 개선 가이드

## 🎯 최적 사용 패턴을 위한 프롬프트 작성법

### 1. 프로젝트 시작 프롬프트
```
프로젝트: [프로젝트명]
/flow [프로젝트명]으로 전환 후 작업 시작

주요 작업:
1. [작업1]
2. [작업2]

워크플로우로 진행 상황 추적하면서 작업해주세요.
```

### 2. 코드 분석 및 수정 프롬프트 (권장)
```
다음 파일들을 분석하고 개선해주세요:
- [파일1.py]
- [파일2.py]

분석 기준:
- TODO 항목 확인
- 중복 코드 찾기
- 성능 개선 가능 부분

parse_file()과 replace_block()을 활용해서 안전하게 수정해주세요.
각 수정 전에 확인 받고, Git 백업도 해주세요.
```

### 3. 연속 작업 프롬프트 (stdout 체인 활용)
```
다음 작업을 단계별로 진행해주세요:

Phase 1: 프로젝트 구조 분석
- scan_directory()로 전체 구조 파악
- 주요 파일 식별

Phase 2: 코드 품질 검사
- parse_file()로 각 파일 분석
- 문제점 리스트 작성

Phase 3: 개선 작업
- 문제점별로 수정 계획 수립
- replace_block()으로 안전하게 수정

각 Phase 완료 시 [NEXT_ACTION] 태그로 다음 작업 지시하고,
task_context 변수에 상태를 저장해서 이어갈 수 있게 해주세요.
```

### 4. Git 통합 작업 프롬프트
```
작업 시작 전 git_status()로 현재 상태 확인하고,
주요 변경사항마다 의미있는 커밋을 만들어주세요.

커밋 규칙:
- feat: 새 기능
- fix: 버그 수정
- refactor: 리팩토링
- docs: 문서 수정

작업 완료 후 전체 변경사항 요약해주세요.
```

### 5. 효율적인 검색 프롬프트
```
다음 패턴들을 코드에서 찾아주세요:
- "TODO" 또는 "FIXME" 주석
- try-except 없는 위험한 코드
- 하드코딩된 값들

search_code()를 사용해서 효율적으로 검색하고,
결과를 카테고리별로 정리해주세요.
```

## 🚀 권장 작업 흐름

### 시작
```
fp("프로젝트명")
workflow("/start 작업명")
show_history()  # 이전 작업 확인
```

### 분석
```
parsed = parse_file("main.py")
# 안전한 dict 반환 - 함수, 클래스 정보
```

### 수정
```
replace_block("file.py", old_code, new_code)
# 자동 백업 생성
```

### 완료
```
git_add(".") 
git_commit("작업 내용")
workflow("/complete 작업 요약")
```

## 💡 고급 팁

1. **변수로 상태 관리**
   - task_context, analysis_result 등 딕셔너리 활용
   - 여러 execute_code 호출 간 데이터 공유

2. **stdout 활용**
   - [NEXT_ACTION]: 다음 작업 지시
   - [CONTEXT]: 참조할 변수
   - [STATUS]: 현재 진행 상황

3. **히스토리 활용**
   - add_history()로 중요 작업 기록
   - continue_from_last()로 이전 작업 재개

4. **안전한 함수 우선**
   - parse_file, search_code, git_status 등
   - 모든 반환값이 dict/list[dict]로 일관성 있음


## 🏗️ 체계적인 개발 프로세스 프롬프트

### 1. 상세설계 요청 프롬프트 (필수)
```
[작업명]에 대한 상세설계를 먼저 제시해주세요.

설계 포함사항:
1. 현재 상태 분석
2. 변경이 필요한 모듈/함수 목록
3. 각 변경사항의 구체적인 방법
4. 예상되는 영향 범위
5. 작업 순서 및 단계

설계 완료 후 "이대로 진행해도 될까요?" 확인 부탁드립니다.
```

### 2. 코드 수정 프롬프트 (승인 프로세스 포함)
```
다음 파일들을 수정해주세요: [파일 목록]

수정 프로세스:
1. 먼저 parse_file()로 현재 코드 구조 분석
2. "## 상세 설계" 섹션으로 다음 내용 제시:
   - 현재 구현 상태
   - 수정할 함수/클래스 목록
   - 각 항목별 변경 내용
   - 변경 이유 및 기대 효과
3. "이 계획대로 진행할까요?" 승인 요청
4. 승인 후 replace_block()으로 단계별 수정
5. 각 단계마다 진행 상황 보고

Git 백업도 잊지 말아주세요.
```

### 3. 단계별 승인 프롬프트
```
복잡한 작업은 단계별로 승인받으며 진행해주세요:

Phase 1: 분석 및 설계
- 현재 코드 분석 결과 제시
- 개선 방안 상세 설계
- ✋ "Phase 1 설계를 검토해주세요" 

Phase 2: 핵심 기능 수정
- 수정할 함수 목록과 변경 내용 제시
- ✋ "다음 함수들을 수정해도 될까요?"
- 승인 후 수정 진행

Phase 3: 테스트 및 검증
- 변경사항 테스트 계획 제시
- ✋ "테스트를 진행할까요?"

각 Phase 후 상세 보고서 제공해주세요.
```

### 4. 결과 보고 프롬프트

```
   - 변경 이유: [왜 변경했는지]
   - 영향 범위: [다른 부분에 미치는 영향]

2. **module2.Class1**
   - 추가된 메서드: [메서드명]
   - 기능: [무엇을 하는지]

### 주요 개선사항
- 성능: [개선된 부분]
```

## 🎯 권장 작업 흐름 (개선판)

### 1️⃣ 시작 및 분석
```python
fp("프로젝트명")
workflow("/start 작업명")

# 현재 상태 분석
parsed = parse_file("main.py")
print("## 현재 코드 분석 결과")
print(f"- 함수: {len(parsed['functions'])}개")
print(f"- 클래스: {len(parsed['classes'])}개")
```

### 2️⃣ 상세 설계 제시
```python
print("""
## 상세 설계

### 1. 현재 상태
- main() 함수: 200줄 (너무 김)
- 에러 처리 부족

### 2. 개선 계획
- main() → 3개 함수로 분리
  - initialize(): 초기화
  - process(): 처리
  - cleanup(): 정리
- try-except 추가

### 3. 작업 순서
1. 백업 생성
2. main() 함수 분리
3. 에러 처리 추가
4. 테스트

이대로 진행해도 될까요?
""")
```

### 3️⃣ 승인 후 수정
```python
# Git 백업
git_commit("backup: 작업 시작 전 백업")

# 단계별 수정
print("🔧 Step 1/3: initialize() 함수 추출")
replace_block("main.py", old_code, new_code)
print("✅ Step 1 완료")

print("🔧 Step 2/3: process() 함수 추출")
# ... 계속
```

### 4️⃣ 결과 보고
```python
print("""
## 📊 작업 결과 보고서

### 변경된 파일
- [x] main.py - main() 함수를 3개로 분리
- [x] utils.py - 에러 처리 헬퍼 추가

### 변경된 함수 상세
1. **main() → initialize(), process(), cleanup()**
   - 변경 내용: 200줄 함수를 3개로 분리
   - 변경 이유: 단일 책임 원칙 적용
   - 영향 범위: 없음 (동일한 동작)

2. **utils.safe_execute() 추가**
   - 기능: 안전한 함수 실행 래퍼
   - 사용처: 모든 외부 API 호출

### Git 커밋
- backup: 작업 시작 전 백업
- refactor: main() 함수를 3개로 분리
- feat: 에러 처리 헬퍼 추가
""")

workflow("/complete 리팩토링 완료")
```

## ⚠️ 중요 체크포인트

### 승인이 필요한 시점
1. **설계 단계** - "이대로 진행해도 될까요?"
2. **대규모 수정 전** - "다음 함수들을 수정해도 될까요?"
3. **삭제 작업 전** - "이 코드를 삭제해도 될까요?"
4. **구조 변경 전** - "폴더 구조를 변경해도 될까요?"

### 보고가 필요한 내용
1. **변경된 모든 파일 목록**
2. **수정/추가/삭제된 함수/클래스**
3. **변경 이유와 기대 효과**
4. **테스트 결과**
5. **Git 커밋 내역**

## 💡 프롬프트 예시 (전체 흐름)

```
프로젝트의 인증 모듈을 개선해주세요.

작업 전에:
1. 현재 인증 모듈 구조를 분석해주세요
2. 개선이 필요한 부분과 방법을 상세설계로 제시해주세요
3. "이대로 진행해도 될까요?" 확인 부탁드립니다

승인 후:
1. Git 백업 먼저 해주세요
2. 각 단계마다 무엇을 수정하는지 알려주세요
3. 중요한 변경은 다시 확인받아주세요

완료 후:
1. 변경된 모든 모듈/함수를 보고해주세요
2. 테스트 결과도 포함해주세요
3. Git 커밋 내역도 정리해주세요
```
